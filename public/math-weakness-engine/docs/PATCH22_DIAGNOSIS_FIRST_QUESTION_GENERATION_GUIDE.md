# Patch22 Guide · Diagnosis-first, optional question generation

## 왜 분리했는가
기존 구조에서는 1차 진단과 10문항/PDF 생성이 한 번에 실행되어, 입력 자료의 단원과 다른 fallback 문제지가 섞일 수 있었습니다. 특히 대수 거듭제곱근 자료를 넣었는데 유리수·무리수 샘플 문제지가 출력되는 문제가 발생했습니다.

## 변경 후 흐름
1. 1차 진단: 자료 유형 판별, 단원/개념 매칭, 학생 문제점, 연결 단원 출력
2. 2차 보강 문제 생성: 사용자가 버튼을 눌렀을 때만 10문항 생성
3. PDF 생성: 10문항 생성 후 학생용 PDF 열기/저장
4. 재검수: 학생이 풀어온 답안을 업로드해 검수

## 핵심 안전장치
- Analyze 단계에서 `state.verificationSet`을 항상 초기화합니다.
- Generate 단계에서 `engine_locked_context`를 payload에 포함합니다.
- Worker prompt에서 이전 학생/이전 단원/fallback 문제 재사용 금지를 명시합니다.
- UI 버튼도 1차 진단 전에는 보강 문제 생성과 PDF를 비활성화합니다.

## 기대 효과
- 분석만 필요한 학생에게 불필요한 문제지를 주지 않습니다.
- 문제가 필요한 학생은 진단 결과에 맞는 문제지만 따로 생성합니다.
- 출력 단원이 진단 단원과 어긋날 가능성을 줄입니다.
