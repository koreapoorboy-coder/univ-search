# Patch23 · 1차 진단 구체화 패치

작성일: 2026-07-09
키워드: math weakness engine / first diagnosis precision / power root fallback / student action result

## 목적

Patch22에서 1차 진단과 2차 보강 문제 생성을 분리한 뒤, 1차 진단 화면이 너무 일반적으로 보이는 문제가 있었다.
예를 들어 `260706 대수 거듭제곱 취약파트.pdf`를 넣었을 때 학생 화면이 다음처럼 나왔다.

- 풀이 과정 진단
- 풀이가 틀어진 위치 확인 필요
- 정밀 분석 필요
- 조건 해석 → 식 세우기 → 계산 전개 → 답 검산

이 구조는 방향은 맞지만 학생이 바로 무엇을 해야 하는지 알기 어렵다. Patch23은 1차 진단 화면에서 확정 단원과 실제 행동 기준을 더 구체적으로 보여주도록 수정한다.

## 수정 핵심

### 1. 파일명/단원명 기반 fallback 주제 감지 강화

Worker가 OpenAI 오류 또는 stub/fallback으로 내려가더라도 파일명, 단원명, lesson_title, teacher_focus를 확인해 다음 주제를 감지한다.

- 거듭제곱 / 거듭제곱근 / n제곱근 / 세제곱근 / 네제곱근 / 유리수 지수 / 분수 지수
- 유리수 / 무리수 / 순환소수 / 분수 꼴 / 제곱근과 실수

특히 `거듭제곱 취약파트`처럼 파일명에 `거듭제곱근`이 완전하게 들어있지 않아도 거듭제곱근 계열로 안전하게 감지한다.

### 2. 거듭제곱근 1차 진단 전용 문장 추가

거듭제곱근 자료가 감지되면 1차 진단 결과는 다음 관점으로 나온다.

- n의 짝수/홀수 여부
- 밑의 부호
- 실수 거듭제곱근 존재 여부와 개수
- √ 기호의 주값과 x^n=a의 모든 해 구분
- 근호식·유리수 지수 계산 후 원래 식 검산

### 3. 학생용 화면에 “학생이 바로 할 일” 추가

기존에는 문제점과 연결 단원만 보였다. Patch23에서는 1차 진단 화면에 학생이 바로 수행할 행동을 3개까지 추가한다.

예: 거듭제곱근 자료

1. 오답 문항마다 n의 짝수/홀수 여부와 밑의 부호를 먼저 표시한다.
2. √ 기호의 주값을 묻는지, x^n=a의 모든 실수해를 묻는지 문제 문장을 표시한다.
3. 계산한 값을 원래 식에 다시 대입해 부호·개수·값을 검산한다.

### 4. 2차 문제 생성 분리 구조 유지

Patch23은 1차 진단 화면만 구체화한다. Patch22의 핵심 계약은 그대로 유지한다.

- 1차 진단에서 10문항 자동 생성 금지
- [보강 문제 10문항 생성] 버튼을 눌렀을 때만 2차 문제 생성
- PDF는 2차 문제 생성 이후에만 가능
- 이전 유리수·무리수 샘플 문제 또는 이전 학생 PDF 재사용 금지

## 테스트 순서

1. GitHub에 Patch23 ZIP의 파일을 덮어쓴다.
2. Cloudflare Worker는 `worker_skeleton/math_diagnosis_worker.js` 내용을 Worker 코드에 적용한다.
3. 사이트에서 `260706 대수 거듭제곱 취약파트.pdf`를 업로드한다.
4. 1차 진단만 실행한다.
5. 확인할 것:
   - 10문항이 자동 생성되지 않는가
   - 상단 결과가 `거듭제곱근 조건 판정과 주값/해 구분 확인 필요`처럼 구체적으로 나오는가
   - 지금 문제점이 거듭제곱근 조건, 주값/모든 해, 근호식·유리수 지수 검산 중심으로 나오는가
   - “학생이 바로 할 일” 3개가 보이는가
6. [보강 문제 10문항 생성]을 누른다.
7. 확인할 것:
   - 거듭제곱근 전용 10문항만 생성되는가
   - 0.5, -3, 0.333... 같은 유리수·무리수 샘플 문항이 나오지 않는가
