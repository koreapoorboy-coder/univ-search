# MINI 전달용 도서 연결 인계 문서

## 목적

210권 도서 데이터를 5번 도서 추천 단계에 연결한다.  
단, 5번 도서 추천은 독립 검색이 아니라 기존 탐구 흐름의 마지막 단계다.

## 반드시 유지할 흐름

```txt
과목 선택
→ 학과 선택
→ 3번 교과 개념 선택
→ 추천 키워드 선택
→ 4번 후속 연계축 확정
→ 5번 도서 추천
```

## 금지 사항

- 학과 단독 도서 추천 금지
- 도서 일반 검색형 추천 금지
- 선택 과목만으로 도서 출력 금지
- 기존 3번 추천 키워드 로직 수정 금지
- 기존 4번 후속 연계축 로직 수정 금지
- 직접 일치 도서 / 확장 참고 도서 UI 구조 변경 금지

## 5번 도서 추천 구조

### 직접 일치 도서
선택한 개념, 추천 키워드, 후속 연계축과 직접 연결되는 책.

### 확장 참고 도서
직접 일치 도서보다 우선순위는 낮지만, 보고서 확장·비교·사회적 의미 부여에 도움이 되는 책.

## 도서 카드에 필요한 필드

```js
{
  bookId,
  sourceId,
  managementNo,
  title,
  author,
  directMatchReason,
  expansionReason,
  inheritedSubject,
  inheritedDepartment,
  inheritedConcept,
  inheritedKeyword,
  inheritedAxis,
  reportUse,
  relatedSubjects,
  relatedMajors,
  sourcePdf,
  sourcePages
}
```

## 추천 결과에 반드시 표시할 근거

- 어떤 선택 개념과 연결되는지
- 어떤 추천 키워드와 연결되는지
- 어떤 4번 후속 연계축과 연결되는지
- 직접 일치인지 확장 참고인지
- 보고서에서 어떤 역할을 하는지

## 상품화 관점

학생들이 책을 잘 읽지 않기 때문에 도서 추천은 단순 목록이 아니라 “보고서 근거와 확장 프레임”으로 작동해야 한다.  
따라서 책은 목적지가 아니라 보고서를 설계하는 근거 자료로 보여줘야 한다.
