# 210권 도서 PDF 1차 통합 정리 결과

## 1. 완료 상태

- 총 처리 도서: **210권**
- 1권: **70권** / 관리번호 1~70
- 2권: **70권** / 관리번호 71~140
- 3권: **70권** / 관리번호 141~210

## 2. 무결성 점검

- 누락 관리번호: `없음`
- 중복 관리번호: `없음`
- 검수 이슈 후보: **16건**

## 3. 현재 처리 성격

현재 통합본은 **적용용 패치가 아니라 Stage1 누적 마스터 + Stage2 인계 파일**이다.

즉, 지금 바로 엔진 파일을 덮어쓰는 단계가 아니다.

- 신규 도서 무단 추가 금지
- 기존 카드 삭제 금지
- 기존 3번 교과 개념·추천 키워드 로직 유지
- 기존 4번 후속 연계축 payload 유지
- 5번 도서 추천은 기존 payload를 상속해서 연결

## 4. 핵심 주의사항

3권의 경우 작업 관리번호는 141~210이지만, 기존 엔진 source catalog 후보는 별도 번호 체계를 가진다.  
따라서 최종 적용 패치에서는 **관리번호와 source_id를 분리**해야 한다.

## 5. 다음 단계

1. 제목·저자·source_id 보정 후보 검수
2. 210권 통합 source master 생성
3. 5번 도서 추천용 matching index 생성
4. 기존 3번·4번 payload를 받아 도서 추천으로 넘기는 adapter 추가
5. UI 구조 유지 상태에서 5번 도서 카드만 연결
6. 테스트 케이스로 과목 → 학과 → 개념 → 추천 키워드 → 후속 연계축 → 도서 추천 흐름 확인
