# REPORT_SEED_BANK_STAGE1_V1

## 0. 문서 목적

이 문서는 `report_dataset_stage1_RPT001_005.xlsx`에 들어 있는 RPT-001~RPT-005 보고서를 **운영 엔진에 적용하지 않고**, 먼저 seed 데이터로만 변환한 파일이다.

핵심은 보고서 원문을 복사하는 것이 아니라, 좋은 보고서가 만들어지는 **사고 구조**를 추출하는 것이다.

```text
학생 선택값
→ seed 패턴 매칭
→ 융합 주제 재조합
→ 섹션별 작성 지시문 구성
→ MINI 보고서 생성
```

현재 산출물은 적용 패치가 아니다.  
운영 중인 `keyword-engine`에는 아직 연결하지 않는다.

---

## 1. 산출 파일

```text
report_seed_bank_stage1_v1.json
report_seed_bank_stage1_v1.js
REPORT_SEED_BANK_STAGE1_V1.md
```

- `.json`: 데이터 검수·수정용 원본 seed bank
- `.js`: 이후 엔진에 연결하기 쉬운 브라우저/Node 겸용 데이터 파일
- `.md`: 작업자/새 채팅 인계용 설명 문서

---

## 2. 전역 사용 원칙

- seed 원문을 복사하지 말고 사고 구조만 사용한다.
- 학생 선택값을 단순 이어붙이지 말고 문제 상황→교과 원리→분석 기준→해결 방향 순서로 재구성한다.
- 참고문헌, 수치, 사례는 지어내지 않는다. 확인되지 않은 자료는 학생 확인 필요로 표시한다.
- 도서는 독후감 소재가 아니라 해석 렌즈·근거·관점 확장 장치로 사용한다.
- 학생 화면에는 seed, RPT 코드, 내부 패턴명 같은 개발자 용어를 노출하지 않는다.
- 현재 파일은 적용용 패치가 아니라 seed 누적용 데이터다.

---

## 3. seed 매칭 우선순위

- 1. 선택 교과/과목과 seed의 bestForSubjects 일치도
- 2. 선택 후속 연계축과 axisTriggers 일치도
- 3. 추천 키워드와 sourceKeywords/axisTriggers 일치도
- 4. 학과/진로와 bestForMajors 일치도
- 5. 보고서 라인(확장형/심화형)과 seed의 분석 깊이 적합도

## 4. Stage 1 seed 목록

### RPT-001 · 시스템·데이터 기반 위험 예측형

- 원 보고서: IoT와 AI를 활용한 지반 붕괴 사고 예방 시스템 구축
- 학생용 표시명: 자료와 기준으로 위험 요인을 예측하고 예방 방안을 설계하는 탐구
- 융합 방식: 교과 원리 + 위험 변수 분해 + 센서/데이터 + 예방 시스템 설계
- 핵심 전개: 현상 관찰 → 사고/위험 요인 분해 → 핵심 변수와 임계값 설정 → 데이터 수집·판단 → 자동화 대응 방안 → 한계와 후속 탐구
- 적합 과목: 물리학, 지구과학, 정보, 수학, 통합과학
- 적합 계열: 토목공학, 건축공학, 건설안전공학, 안전공학, AI, 데이터과학, 공간정보공학
- 축 트리거: 데이터 해석, 모델링, 시스템 설계, 위험 예측, 안전 관리, 센서, 자동화, 공학 적용
- 주제 합성 공식: 교과 원리로 위험 원인을 설명하고, 측정 가능한 변수와 임계값을 세운 뒤, 데이터 기반 예방 시스템을 제안한다.

### RPT-002 · 지속가능성·소재·디자인 해결형

- 원 보고서: 지속가능 패션을 위한 가시적 수선과 업사이클링
- 학생용 표시명: 생활 속 제품 문제를 소재 원리와 지속가능성 관점으로 해결하는 탐구
- 융합 방식: 소재 물성 + 자원순환 + 디자인/소비자 행동 + 환경 영향
- 핵심 전개: 생활 문제 인식 → 환경·사회적 비용 확인 → 소재/구조 원리 설명 → 대안 설계 → 효과 비교 → 실천 가능성 평가
- 적합 과목: 통합과학, 물리학, 수학, 환경, 미술/디자인
- 적합 계열: 섬유공학, 의류학, 패션디자인, 환경공학, 지속가능디자인, 산업디자인
- 축 트리거: 지속가능성, 자원순환, 업사이클링, 소재 물성, 생활 문제, 디자인, 환경 영향, 소비자
- 주제 합성 공식: 생활 속 문제를 소재 특성으로 분석하고, 지속가능한 대안 설계를 통해 환경적·사회적 효과를 평가한다.

### RPT-003 · 지구시스템·기후 데이터 해석형

- 원 보고서: 우리 동네 기후의 미래는 바다가 결정한다 — 해수 순환과 한반도 기후변화의 연결고리
- 학생용 표시명: 지구 시스템의 변화가 지역 환경에 미치는 영향을 자료로 해석하는 탐구
- 융합 방식: 지구과학 원리 + 기후 시나리오 + 지역 영향 + 자료 해석
- 핵심 전개: 지역 현상 의문 → 지구 시스템 원리 확인 → 장기 변화 자료 탐색 → 시나리오 비교 → 지역 영향 해석 → 후속 관측/분석 제안
- 적합 과목: 지구과학, 통합과학, 수학, 확률과 통계, 환경
- 적합 계열: 지구과학, 대기과학, 해양학, 환경공학, 기후과학, 데이터과학
- 축 트리거: 기후변화, 해양 순환, 시나리오, 장기 데이터, 지역 영향, 자료 해석, 환경 변화
- 주제 합성 공식: 지구 시스템 원리를 바탕으로 장기 자료와 시나리오를 비교하여 지역 환경 변화의 원인과 영향을 설명한다.

### RPT-004 · 사회문제·정책 거버넌스 평가형

- 원 보고서: 국립대 통합 거버넌스와 지방행정 위기 극복 방안
- 학생용 표시명: 사회 문제를 이해관계자·정책 기준·효과 평가로 분석하는 탐구
- 융합 방식: 사회 문제 + 제도/정책 + 이해관계자 + 비용·편익 또는 효과 평가
- 핵심 전개: 사회 위기 진단 → 이해관계자 구조 파악 → 정책/제도 개념 설명 → 평가 기준 설정 → 사례 비교 → 개선 방안 제안
- 적합 과목: 통합사회, 사회·문화, 정치와 법, 경제, 한국지리
- 적합 계열: 행정학, 정책학, 지방자치, 공공정책, 교육행정, 도시행정
- 축 트리거: 정책, 거버넌스, 지방, 공공성, 제도, 비용편익, 사회문제, 이해관계자
- 주제 합성 공식: 사회 문제를 이해관계자와 정책 기준으로 나누어 분석하고, 효과와 한계를 비교해 개선 방향을 제시한다.

### RPT-005 · 공간 데이터·도시 최적화 모델링형

- 원 보고서: 진주시 AI 대전환 기반 스마트시티 공간 최적화
- 학생용 표시명: 공간 자료와 모델을 활용해 도시 문제의 효율성과 형평성을 함께 분석하는 탐구
- 융합 방식: 공간 구조 + 수학 모델 + 데이터 시각화 + 도시 문제 해결
- 핵심 전개: 지역 문제 인식 → 공간 변수 설정 → 수학/지리 모델 적용 → 효율성·형평성 비교 → 최적화 방안 제시 → 확장 가능성 검토
- 적합 과목: 한국지리, 확률과 통계, 수학, 물리학, 정보
- 적합 계열: 도시계획학, 도시공학, 교통공학, 스마트시티, 공간정보공학, 지리학
- 축 트리거: 공간, 도시, 스마트시티, 입지, 접근성, 모델링, 데이터, 형평성, 최적화
- 주제 합성 공식: 지역 문제를 공간 변수로 바꾸고, 수학·지리 모델로 비교해 효율성과 형평성을 함께 고려한 해결 방향을 제안한다.

## 5. 14개 내부 섹션 → 8개 학생 출력 섹션 변환

내부 seed에는 원 보고서의 14개 섹션 구조를 보존한다.  
하지만 학생 화면에는 14개를 그대로 노출하지 않고 아래 8개 구조로 재조합하는 것이 기준이다.

| 학생 출력 섹션 | 사용 역할 |
|---|---|
| 1. 탐구 주제와 핵심 질문 | 제목, 중요성, 핵심 질문 |
| 2. 탐구 동기 | 교과 수업의 의문 → 실제 문제 → 진로 관심 |
| 3. 핵심 개념과 원리 | 관련 키워드, 개념 설명 |
| 4. 해결할 문제와 중요성 | 문제 상황, 해결 필요성 |
| 5. 실제 적용 및 문제 해결 과정 | 자료 수집, 기준 설정, 분석, 대안 제시 |
| 6. 교과목 연계 | 선택 과목, 횡단 연결, 종단 연결 |
| 7. 심화 탐구 발전 방향 | 후속 탐구, 필요한 자료, 확장 가능성 |
| 8. 세특 문구와 참고자료 | 학생부 문장, 참고자료, 학생 확인 필요 사항 |

## 6. 다음 단계 제안

1. 코치님이 가진 보고서 파일을 계속 같은 방식으로 추가한다.
2. seed가 20~30개 이상 쌓이면 패턴명을 재분류한다.
3. 그 뒤에만 `report_seed_matcher.js`를 만들고 엔진에 연결한다.
4. 운영 반영 시에는 먼저 관리자/테스트 화면에서만 seed 매칭 결과를 확인한다.
5. 정상 확인 후 학생용 2차 업데이트에 반영한다.
