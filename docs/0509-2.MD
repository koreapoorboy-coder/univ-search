# 1. 절대 지켜야 할 최우선 원칙

## 1-1. 학과명은 반드시 실제 데이터에 있는 학과명만 사용할 것

앞으로 패치/검수 안내에서 학과명을 제시할 때는 반드시 **사용자가 업로드한 `keyword-engine.zip` 내부 데이터에 실제 존재하는 학과명만** 사용해야 한다.

임의로 학과명을 만들거나, 일반적으로 있을 법한 학과명을 예시로 넣으면 안 된다.

금지 예시:

```text
재료공학과
나노소재공학과
도시공학전공
융합AI학과
문화콘텐츠학부
```

위와 같은 학과가 실제 ZIP 데이터에 없으면 절대 검수 안내에 쓰지 않는다.

반드시 먼저 ZIP 내부에서 실제 존재 학과명을 확인한 뒤 작업한다.

작업 순서:

```text
1. ZIP 내부 실제 학과명 확인
2. 실제 존재 학과명만 기준으로 대표 학과 선정
3. 실제 존재 학과명만 기준으로 표본 확인 학과 선정
4. 데이터에 없는 학과명은 예시로도 사용하지 않음
```

---

## 1-2. 3번 교과 개념도 반드시 실제 데이터에 있는 값만 사용할 것

3번 교과 개념은 설명용 표현으로 바꾸면 안 된다.

반드시 실제 화면 또는 실제 데이터에 존재하는 3번 교과 개념명을 그대로 사용해야 한다.

예를 들어 아래처럼 임의 변환하면 안 된다.

```text
산염기 반응
원자 구조 해석
화학 반응 속도
반도체 소재 분석
```

실제 데이터에 있는 값만 사용한다.

예시:

```text
화학 반응에서의 동적 평형
현대의 원자 모형과 전자 배치
원소의 주기적 성질
화학 결합
분자의 구조와 성질
```

---

## 1-3. 4번 후속 연계축도 반드시 실제 데이터에 있는 축명만 사용할 것

4번 축은 가장 자주 오류가 났던 부분이다.

절대 설명용 축명을 임의로 만들면 안 된다.

실제 화면에 나오거나 실제 JSON/JS 데이터에 존재하는 축명만 기준으로 도서 매칭을 해야 한다.

오류 사례:

```text
원자 구조·전자 배치 해석 축
```

이 축은 화학+컴퓨터공학과 실제 화면값이 아니었다.
실제 화학+컴퓨터공학과에서는 아래처럼 나와야 했다.

```text
전자배치·스펙트럼 축
에너지 준위·빛 해석 축
소재·전자 구조 응용 축
```

따라서 새 채팅 작업자는 반드시 아래 순서를 지킨다.

```text
1. 실제 3번 교과 개념 확인
2. 해당 3번에서 실제로 뜨는 4번 축 확인
3. 실제 4번 축명 기준으로만 5번 도서 매칭 수정
4. 4번 축명을 임의로 새로 만들지 않음
```

# 2. 계열 묶음 패치 안내 방식

앞으로 계열 묶음 패치를 줄 때는 전체 학과를 모두 확인하라고 하면 안 된다.

반드시 아래 구조로 안내한다.

```text
1. 대표로 전체 확인할 학과 1개
2. 대표 학과에서 전체 확인할 3번·4번·5번 로직
3. 나머지 계열 학과는 표본 확인할 핵심 축 2~3개만 제시
4. 정상 기준 / 비정상 기준 제시
```

대표 확인 방식:

```text
대표 학과 1개 = 전체 3번·4번·5번 확인
나머지 같은 계열 학과 = 핵심 축 2~3개만 표본 확인
완전히 다른 계열 = 새로 대표 학과 잡고 전체 확인
```

이 규칙은 반드시 유지한다.

# 3. 작업 철학

이 엔진은 단순한 키워드 추천기나 도서 추천기가 아니다.

핵심 구조는 아래와 같다.

```text
희망 진로/학과
→ 과목 선택
→ 교과 개념 선택
→ 추천 키워드 선택
→ 교과 개념 기반 후속 연계축 선택
→ 도서 추천
→ 보고서 방향 설계
→ mini payload 전달
→ 최종 보고서 생성
```

도서는 독후감용이 아니라, 탐구 보고서의 **근거·관점·확장성**을 제공하는 장치다.

따라서 5번 도서 추천은 항상 아래 구조를 따라야 한다.

```text
3번 교과 개념
→ 4번 후속 연계축
→ 직접 도서
→ 확장 도서
→ 추천 이유
→ 보고서 활용 문장
```

# 6. 현재 남은 작업

현재 남은 큰 덩어리는 아래다.

```text
인문계열
사회과학계열
상경계열
```

정확히는 “인문계만”이라기보다 다음 세 묶음으로 보는 것이 안전하다.

```text
1. 인문계열
2. 사회과학계열
3. 상경계열
```

다음 작업에서 반드시 먼저 해야 할 일:


```text
수행평가 탐구엔진 keyword-engine 도서 5번 검수 작업을 이어서 진행한다.

현재 v156까지 완료되었고, 이공계·자연계·공학계열 핵심 도서 매칭은 1차 잠금 완료로 본다.

반드시 지켜야 할 원칙:
1. 학과명은 반드시 업로드한 keyword-engine.zip 내부 실제 데이터에 존재하는 학과명만 사용한다.
2. 3번 교과 개념은 실제 데이터에 있는 값만 사용한다.
3. 4번 후속 연계축도 실제 화면 또는 실제 데이터에 존재하는 축명만 사용한다.
4. 임의 학과명, 임의 교과 개념명, 임의 축명은 예시로도 사용하지 않는다.
5. 계열 묶음 패치 시에는 대표 학과 1개만 전체 확인하고, 나머지는 실제 존재 학과명 기준으로 핵심 축 2~3개만 표본 확인한다.
6. ACCESS_GATEWAY, mini worker, 보고서 생성 구조, UI 구조는 건드리지 않는다.
7. 수정은 book_210_ui_bridge.js 중심의 최소 패치로 진행한다.

먼저 ZIP 내부에서 실제 존재하는 인문·사회·상경 학과명과 관련 3번·4번 데이터를 확인한 뒤, 대표 확인 학과 1개와 표본 확인 학과를 제시해줘.
```

# 10. 다음 작업자 주의사항
