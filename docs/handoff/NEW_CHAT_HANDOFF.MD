# NEW_CHAT_HANDOFF_2026-04-28

## 0. 새 채팅 시작 시 가장 먼저 읽을 문장
이 프로젝트는 `keyword-engine`의 **교과 개념 기반 수행평가 탐구엔진**이다. 현재 작업은 3번 `교과 개념·키워드 선택`과 4번 `교과 개념 기반 후속 연계축 선택`을 과목별·학과별·키워드별로 정교화하는 단계였고, 2026-04-28 기준으로 업로드된 교과에 대한 3번·4번 로직은 1차 완료되었다.

새 채팅에서 절대 하면 안 되는 것:
- 기존 3번·4번 구조를 다시 만들지 않는다.
- UI를 quickstart/hybrid/compact 구조로 바꾸지 않는다.
- 5번 도서, 6번 보고서 방식, 7번 관점, 8번 보고서 라인을 기존 3번·4번 payload와 분리해서 새로 추천하지 않는다.
- 실제 등록 학과가 아닌 임의 학과명을 검수 기준으로 쓰지 않는다.
- 오래된 패치 zip을 최신 로컬 프로젝트 위에 다시 덮어쓰지 않는다.

새 채팅에서 해야 할 것:
- 사용자가 최신 `keyword-engine.zip`을 주면, 그것을 기준선으로 삼는다.
- 3번·4번은 이미 1차 잠금된 상태로 보고, 다음 작업은 5번 도서 연결부터 진행한다.
- 단, 새 zip이 구버전인지 의심되면 `textbook_concept_helper.js`, `subject_concept_ui_seed.json`, `subject_concept_engine_map.json`, `followup-axis` 파일 상태를 먼저 확인한다.
- 모든 다음 단계는 `selectedSubject → selectedMajor → selectedConcept → selectedKeyword → selectedFollowupAxis` 흐름을 그대로 상속해야 한다.

사용자가 유지하기로 한 화면 흐름은 아래다.

- 다음 작업의 핵심은 5번 이후를 기존 구조에 **추가 연결**하는 것이다.

## 3. 반드시 유지해야 하는 정책

### 3-1. 교과 개념·키워드·후속축 정책
아래 정책은 세포와 물질대사에서 명시적으로 반영했고, 이후 모든 과목·5번 이후 단계에도 동일하게 적용해야 한다.

```json
{
  "policy": {
    "priority_rule": [
      "먼저 교과 개념별 종단 연결 후보를 제시한다",
      "학과 입력 전에는 가능한 종단축 전체를 보여준다",
      "학과 입력 후에는 축을 제거하지 않고 우선순위만 재정렬한다",
      "키워드는 다음 단계에서 같은 개념 안의 세부 정렬·설명 보정에 사용한다"
    ],
    "keyword_refinement_rule": [
      "추천 키워드는 같은 개념 안에서 4번 후속 연계축의 우선순위를 정교화한다",
      "키워드가 축과 직접 맞으면 1순위 축, 짧은 라벨, 설명 문구, 활동 예시까지 함께 바뀐다",
      "직접 일치가 약한 경우에도 보조축이 살아 있도록 하여 학생 선택 폭을 남긴다"
    ]
  }
}
```

### 3-2. 학과는 우선순위 보정 장치
학과는 추천 순서를 바꾸는 장치다. 학과가 선택지를 삭제하거나 학생 선택을 잠그면 안 된다.

정상 구조:
- 추천 개념 3개 우선 노출
- 다른 교과 개념 전체 보기 가능
- 선택 키워드에 따라 4번 축 분화
- 학과는 4번 축 순서와 라벨을 조정

- 키워드를 무시하고 책 추천
- 4번 후속축을 무시하고 책 추천

해야 하는 방식:

도서 추천은 아래 payload를 받아야 한다.

```json
{
  "selectedSubject": "과목명",
  "selectedMajor": "학과명",
  "selectedConcept": "3번에서 고른 교과 개념",
  "selectedKeyword": "3번 오른쪽에서 고른 추천 키워드",
  "selectedFollowupAxis": "4번에서 선택되거나 1순위로 제시된 후속 연계축",
  "axisLabel": "직접 연계 강함 / 역량 브리지 / 일반 탐구",
  "linkedSubjects": ["후속 과목", "횡단 과목"],
  "activityExample": "4번 카드의 활동 예시",
  "longitudinalPath": "다음 학년 과목 또는 후속 탐구 방향"
}
```

이 payload를 바탕으로 도서는 최소 3종으로 갈라져야 한다.

1. 직접 일치 도서
   - 선택 키워드와 직접 관련 있는 책
2. 확장 참고 도서
   - 4번 후속 연계축과 관련 있는 책
3. 보고서 관점 보강 도서
   - 6번/7번 보고서 방식과 관점 선택에 도움 되는 책

## 7. 다음 작업 시 파일 접근 순서

새 채팅에서 사용자가 최신 프로젝트 zip을 주면 아래 순서로 확인한다.

주의: 현재까지는 3번·4번이 잠금 상태이므로, 5번 이후를 작업할 때 1~7번 파일은 가급적 읽기 기준으로만 보고, 필요할 때만 최소 수정한다.

## 8. 새 채팅 첫 지시문 예시
아래 문장을 새 채팅 첫 메시지에 붙여넣으면 된다.

> 현재 `keyword-engine`은 2026-04-28 기준으로 업로드된 교과의 3번 교과 개념·키워드 선택과 4번 후속 연계축 선택이 1차 잠금된 상태다. 기존 3번·4번 구조는 절대 다시 만들지 말고, 다음 작업은 5번 도서 연결부터 진행한다. 5번 도서 연결은 반드시 `selectedSubject`, `selectedMajor`, `selectedConcept`, `selectedKeyword`, `selectedFollowupAxis`, `axisLabel`, `linkedSubjects`, `activityExample`, `longitudinalPath`를 상속해야 한다. 실제 등록 학과명만 사용하고, 과거 패치 zip을 다시 덮어쓰지 않는다. 최신 사용자의 로컬 프로젝트 zip을 기준선으로 삼아라.

## 9. 최종 인계 결론

현재 프로젝트에서 가장 중요한 기준은 아래다.

> 3번과 4번은 단순 화면 단계가 아니라, 5번 이후 보고서 생성의 핵심 입력값이다.  
> 앞으로 도서·보고서 방식·관점·라인·MINI 생성은 반드시 3번 선택 개념과 4번 후속 연계축을 상속해야 한다.

이 기준을 지키지 않으면 엔진이 요구하는 형태의 보고서가 나오지 않는다.
