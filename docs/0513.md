## 입력칸 자동 힌트

자료 유형을 선택하면 입력칸에 예시 힌트가 바뀌어야 한다.

예:

```text
자료 유형: 통계 자료

자료 핵심 내용 placeholder:
예: 2020년부터 2024년까지 ○○ 값이 증가/감소했다.

내 해석 placeholder:
예: 이 변화는 ○○ 조건에서 차이가 생긴다는 판단 근거가 된다.
```

```text
자료 제목 placeholder:

예: 언론사명, 발행일

자료 핵심 내용 placeholder:
예: 기사에서 제시한 핵심 사례나 수치

내 해석 placeholder:
예: 이 사례는 내가 선택한 기준 중 ○○와 연결된다.
```

# 9. v231에서 mini에 전달해야 할 payload 구조

현재 v230에서 mini 재호출이 들어갔지만, 선택 방향의 중심성이 약하다.
v231에서는 mini 요청 payload에 아래 구조를 명확히 넣어야 한다.

```js
{
  mode: "secondary_expansion_draft",
  primaryDesign: {
    department,
    subject,
    concept,
    keyword,
    followupAxis,
    performanceAssessment,
    reportStructure
  },
  secondaryExpansion: {
    selectedPathId,
    selectedPathLabel,
    selectedPathQuestion,
    selectedPathEvidenceType,
    selectedPathStructure
  },
  studentEvidence: {
    evidenceType,
    title,
    source,
    keyContent,
    interpretation
  },
  generationRules: {
    useSelectedPathAsMainFrame: true,
    doNotRewritePrimaryDesignOnly: true,
    doNotInventEvidence: true,
    produceDraftParagraphs: true,
    markMissingEvidenceAsStudentInputNeeded: true
  }
}
```

핵심은 이거다.

```text
useSelectedPathAsMainFrame: true
doNotRewritePrimaryDesignOnly: true
produceDraftParagraphs: true
```

이 3개가 들어가야 현재 문제를 해결할 수 있다.

# 10. v231 mini 프롬프트 핵심 문구

mini에 보내는 프롬프트에는 아래 문구가 반드시 들어가야 한다.

```text
너는 지금 1차 탐구 설계서를 다시 요약하는 것이 아니다.
학생이 선택한 2차 확장 방향을 중심으로 보고서 초안을 작성해야 한다.

1차 설계값은 배경 정보로만 사용한다.
보고서의 문단 구조, 자료 배치, 분석 방식, 결론 방향은 반드시 selectedExpansionPath를 기준으로 구성한다.

학생이 입력한 자료 제목, 출처, 핵심 내용, 해석이 있으면 해당 자료를 보고서 문단 안에 반영한다.
자료가 비어 있으면 절대 지어내지 말고 [학생 입력 필요]로 표시한다.

출력은 설계서 요약이 아니라 실제 보고서 초안이어야 한다.
단, 완성본 제출용이 아니라 학생이 수정할 수 있는 초안 형태로 작성한다.
```

# 11. 검수 기준

## 대표 확인 1개

현재 사용자가 확인한 테스트 흐름:

```text
과목: 공통수학1
진로/학과: 반도체공학과
교과 개념: 이차방정식과 이차함수
추천 키워드: 그래프 변화·모델링 전공 맞춤 추천 그래프 해석
후속축: 그래프 변화 해석 축
수행평가 영역명: 최댓값 구조·성능·안정성 비교 분석하기
```

현재 화면에서 2차 확장 방향이 정상 출력된다.

```text
원리 분석형
사례 비교형
자료 해석형
사회적 의미형
후속 탐구형
```

## v231 적용 후 대표 검수

### 검수 A: 자료 해석형 선택

```text
2차 확장 방향: 자료 해석형
자료 제목: 예: 통계 자료
출처: 예: KOSIS 국가통계포털
자료 핵심 내용: 예: 가장 중요한 수치 1~2개
내 해석: 예: 이 자료를 보면 어떤 조건에서 차이가 생기는지 판단할 수 있다.
```

정상 기대:

```text
보고서 초안이 자료 해석형 구조로 나와야 한다.
1차 설계서를 다시 요약하면 안 된다.
자료 제목, 출처, 핵심 내용, 내 해석이 본문에 들어가야 한다.
자료가 빈칸이면 [학생 입력 필요]로 표시해야 한다.
```

### 검수 B: 원리 분석형 선택

정상 기대:

```text
보고서 구조가 개념 원리 → 작동 조건 → 사례 적용 → 한계 → 후속 탐구 순서로 나와야 한다.
자료 해석형처럼 통계 중심으로 나오면 안 된다.
```

### 검수 C: 사례 비교형 선택

정상 기대:

```text
보고서 구조가 사례 A/B 비교 중심으로 나와야 한다.
비교 기준이 문단에 분명히 들어가야 한다.
```

# 13. 사용자 선호 작업 방식

코치님 선호:

```text
1. 부분 함수 교체보다 패치 ZIP 선호
2. 파일명은 짧은 영문으로

6. 실제 데이터에 없는 학과명·과목명·개념명은 예시로 쓰지 않음

```
