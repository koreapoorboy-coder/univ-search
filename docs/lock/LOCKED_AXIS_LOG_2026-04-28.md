# LOCKED_AXIS_LOG_2026-04-28

## 0. 문서 목적
이 문서는 `keyword-engine` 프로젝트에서 **현재까지 검수 완료된 교과 개념 → 추천 키워드 → 4번 후속 연계축 로직**을 잠금 기준으로 정리한 문서다.

이번 문서는 단순 작업 요약이 아니라, 이후 작업자가 새 채팅에서 다시 작업할 때 **기존 구조를 깨지 않고 이어서 작업하기 위한 기준 문서**다.

가장 중요한 결론은 아래다.

> 현재 업로드·적용된 교과 기준으로 **1~4단계 중 3번 교과 개념·키워드 선택과 4번 교과 개념 기반 후속 연계축 선택은 1차 완료**다.  
> 앞으로 5번 도서, 6번 보고서 전개 방식, 7번 관점, 8번 보고서 라인, 최종 MINI 보고서 생성은 이 잠금된 3번·4번 payload 위에 **추가 연결**해야 한다.  
> 기존 3번·4번 로직을 새로 만들거나 우회하면 안 된다.

## 1. 현재 잠금 범위

### 1-1. 완료된 흐름
현재 완료된 범위는 아래 흐름이다.

1. 과목 선택
2. 실제 등록 학과 선택
3. 학과별 교과 개념 추천 순서 정렬
4. 개념 선택 후 추천 키워드 노출
5. 추천 키워드 선택
6. 선택 키워드에 따라 4번 후속 연계축 1순위 및 보조축 재정렬
7. 학과와 직접 맞는 축은 `직접 연계 강함`으로 표시
8. 직접 맞지 않지만 탐구 확장에 유의미한 축은 `역량 브리지`, `일반 탐구` 등으로 남김

### 1-2. 완료 판정 기준
다음 조건을 모두 통과한 교과만 잠금 처리했다.

- 학과가 바뀌면 3번 추천 개념 3개가 달라진다.
- 같은 개념 안에서도 추천 키워드가 바뀌면 4번 첫 카드가 달라진다.
- 같은 성격의 키워드는 같은 축으로 묶이되, 다른 성격의 키워드는 다른 축으로 분화된다.
- 학과가 개념을 강제로 잠그지 않고, 다른 교과 개념을 펼쳐볼 수 있는 구조가 유지된다.
- 학과가 4번 축을 완전히 제거하지 않고, 우선순위만 조정한다.
- 실제 등록 학과 기준으로 검수했다.
- 추천 키워드가 다음 단계 보고서 생성에서 사용할 수 있을 만큼 구체적이다.

## 2. 전체 적용 원칙 — 반드시 유지

### 2-1. 기존 작업에 “추가”해야 한다
이 프로젝트에서 앞으로 추가하는 로직은 반드시 기존 3번·4번 결과 위에 붙어야 한다.

금지:
- 5번 도서 단계에서 학과와 과목만 보고 새로 추천하기
- 6번 보고서 전개 방식에서 선택 키워드와 4번 축을 무시하기
- 7번 관점에서 임의로 진로 관점만 뽑기
- 8번 보고서 라인에서 교과 개념과 후속 연계축을 빼고 일반 보고서처럼 만들기
- MINI 생성 payload에서 `selectedConcept`, `selectedKeyword`, `selectedFollowupAxis`를 누락하기

필수:
- 5번 도서 연결은 3번 선택 개념과 추천 키워드, 4번 후속 연계축을 반영해야 한다.
- 6번 보고서 전개 방식은 선택 키워드의 성격에 맞게 바뀌어야 한다.
- 7번 관점은 학과만이 아니라 4번 축의 성격을 받아야 한다.
- 8번 보고서 라인은 “교과 개념 → 키워드 → 후속 연계축 → 도서/자료 → 보고서 방식”의 경로를 유지해야 한다.
- 최종 MINI 보고서 생성은 위 흐름을 payload로 받아야 한다.

### 2-2. 후속 연계축은 종단 연결이다
4번 후속 연계축은 현재 보고서를 꾸미는 장치가 아니다. 핵심은 다음 학년·다음 과목·다음 활동으로 이어지는 종단 구조다.

따라서 4번 축에는 항상 아래 정보가 포함되어야 한다.

- 현재 선택 과목
- 선택 학과
- 선택 교과 개념
- 선택 추천 키워드
- 후속 과목 또는 연결 과목
- 횡단 연결 과목
- 종단 확장 방향
- 활동 예시
- MINI가 사용할 수 있는 축 라벨

### 2-3. 학과는 “선택을 잠그는 장치”가 아니다
학과는 추천 우선순위를 바꾸는 장치다. 학생이 다른 교과 개념을 보고 싶으면 전체 개념을 펼쳐 선택할 수 있어야 한다.

유지해야 할 구조:
- 기본 추천 3개 먼저 노출
- `다른 교과 개념 보기`로 전체 개념 접근 가능
- 선택 키워드에 따라 4번 축 분화
- 학과는 4번 축 순서와 라벨을 조정

### 2-4. 실제 등록 학과명만 사용
검수와 문서 작성에서 실제 엔진에 없는 학과명을 쓰면 안 된다.

이번 작업 중 정정한 사례:

| 잘못 쓰기 쉬운 표현 | 실제 사용해야 하는 학과명 |
|---|---|

초기에는 `생명공학과`가 환경·생태 분기 또는 대사·건강 분기에 잘못 걸렸다. 최종적으로 생명공학과는 `유전자와 염색체 / 생명과학의 이해 / 물질대사와 에너지` 흐름으로 수정했다.

## 5-6. 세포와 물질대사

### 최종 상태
세포와 물질대사는 정책 기준까지 반영하여 잠금 완료했다.

### 반드시 유지해야 하는 정책
```json
{
  "subject_name": "세포와 물질대사",
  "policy": {
    "priority_rule": [
      "먼저 교과 개념별 종단 연결 후보를 제시한다",
      "학과 입력 전에는 가능한 종단축 전체를 보여준다",
      "학과 입력 후에는 축을 제거하지 않고 우선순위만 재정렬한다",
      "키워드는 다음 단계에서 같은 개념 안의 세부 정렬·설명 보정에 사용한다"
    ],
    "keyword_refinement_rule": [
      "추천 키워드는 같은 개념 안에서 4번 후속 연계축의 우선순위를 정교화한다",
      "키워드가 축과 직접 맞으면 1순위 축, 짧은 라벨, 설명 문구, 활동 예시까지 함께 바뀐다",
      "직접 일치가 약한 경우에도 보조축이 살아 있도록 하여 학생 선택 폭을 남긴다"
    ]
  }
}
```

이 정책은 세포와 물질대사에만 국한되는 것이 아니라, 이후 과목 추가 시에도 기본 원칙으로 재사용할 수 있다.

### 핵심 검수 학과
- 생명공학과
- 약학과
- 식품공학과
- 환경공학과
- 컴퓨터공학과

| 학과 | 개념 | 키워드 | 4번 1순위 정상값 |
|---|---|---|---|
| 생명공학과 | 세포의 구조와 물질 이동 | 세포막 | 세포 시스템·물질 이동 해석 축 |

## 7. 다음 단계로 넘어가기 전 체크리스트

다음 단계는 5번 도서 연결, 6번 보고서 방식, 7번 관점, 8번 보고서 라인이다. 이때 반드시 아래를 지켜야 한다.

### 7-1. 5번 도서 연결 필수 조건
도서 추천은 학과와 과목만 보지 말고 아래 정보를 모두 사용해야 한다.

- selectedSubject
- selectedMajor
- selectedConcept
- selectedKeyword
- selectedFollowupAxis
- axisLabel
- linkedSubjects
- longitudinalDirection
- activityExample

### 7-2. 6번 보고서 방식 필수 조건
보고서 방식은 고정 4종이라도, 설명 문구와 추천 순서는 4번 축에 따라 달라져야 한다.

예:
- 데이터 시각화 축 → 자료 분석형, 모델링형 우선
- 실험/측정 축 → 실험보고서형 우선
- 윤리·사회 축 → 쟁점 분석형, 자료조사 보고서형 우선
- 공정/소재 설계 축 → 설계형, 비교 분석형 우선

### 7-3. 7번 관점 필수 조건
관점은 단순히 전공 관점이 아니라, 선택 키워드와 4번 후속 축의 사고 방식을 반영해야 한다.

예:
- `기상 데이터 시각화 축` → 데이터 해석 관점
- `의료 센서·영상 응용 축` → 측정·진단 관점
- `세포 신호·수용체 응용 축` → 생체 조절·응용 관점
- `반도체·소재 설계 축` → 물성·소자 설계 관점

### 7-4. 8번 보고서 라인 필수 조건
최종 보고서 라인은 아래 구조를 유지해야 한다.

1. 현재 교과 개념
2. 선택 키워드
3. 선택 키워드가 여는 후속 연계축
4. 후속 과목 또는 연결 과목
5. 도서/자료 연결
6. 보고서 방식
7. 관점
8. 활동 예시
9. MINI 생성 payload
