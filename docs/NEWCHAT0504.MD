## 2. 지금까지 완료된 핵심 작업

## 2-1. 3번·4번 1차 잠금 완료

현재 완료된 것은 **3번 교과 개념 추천 + 4번 교과 개념 기반 후속 연계축의 1차 정확도 잠금**이다.

이 완료의 의미는 다음과 같다.

| 항목                 | 상태             |
| ------------------ | -------------- |
| 3번 추천 개념 대표 3개     | 계열별 1차 정리 완료   |
| 4번 교과 개념 기반 후속 연계축 | 계열별 큰 오류 수정 완료 |
| 전공별 오인식            | 주요 계열 기준 보정 완료 |
| 추천 키워드 선택 시 후속축 연결 | 1차 정상화 완료      |
| 1순위 라벨 제거          | 완료             |
| 학과 라우팅             | 건드리지 않음        |
| 보고서 구조             | 건드리지 않음        |
| mini worker 구조     | 건드리지 않음        |
| 도서 추천 구조           | 아직 본격 수정 전     |

중요한 해석은 다음과 같다.

```text
지금까지의 잠금은 “완전한 세부 디테일링”이 아니라
계열별로 엉뚱한 교과 개념이나 후속축이 뜨지 않도록 막는 1차 정확도 잠금이다.
```

따라서 2차에서는 추천 키워드별 변별도, mini payload 반영, 보고서 결과문 반영을 확인해야 한다.

## 2-2. 1순위 라벨 제거 완료

4번 후속 연계축에서 `1순위` 표기는 삭제했다.

이유는 다음과 같다.

```text
4번 후속 연계축은 정답형 추천이 아니라 학생 선택형 후보군이다.
1순위라고 표시하면 학생이 정답처럼 느낄 수 있으므로,
내부 정렬은 유지하되 화면 라벨은 제거하는 방향으로 처리했다.
```

현재 기준:

```text
- 내부 점수/정렬 로직은 유지
- 화면의 1순위 라벨은 삭제
- 학생은 여러 후속축 후보 중 직접 선택
```

## 5. 3번·4번 완료 후 다음 단계

지금 상태는 다음과 같다.

```text
3번·4번 1차 정확도 잠금 완료.
다음은 5번 도서 추천 작업으로 넘어간다.
```

단, 5번에 들어가기 전 이해해야 할 구조는 아래와 같다.

## 5-1. 5번 도서의 역할

5번 도서는 3번·4번에서 선택한 값을 받아서 **보고서 근거 도서**를 추천하는 단계다.

연결 흐름:

```text
학과
→ 과목
→ 3번 선택 개념
→ 추천 키워드
→ 4번 후속 연계축
→ 5번 도서 추천
→ 선택 도서
→ mini payload
→ 최종 보고서 반영
```

5번에서 확인해야 할 핵심 전달값:

```text
subject
department
selectedConcept
selectedRecommendedKeyword
followupAxis
selectedBook
bookReason
bookUsage
```

---

## 5-2. 5번 도서는 지금처럼 묶어서 진행해야 한다

3번·4번처럼 모든 학과를 하나씩 보면 안 된다.
5번은 도서 도메인 기준으로 묶어서 가야 한다.

## 5번 1차 도서 묶음

| 도서 묶음  | 대표 검수 학과 | 함께 판단 가능한 계열           |
| ------ | -------- | ---------------------- |
| BOOK-A | 컴퓨터공학과   | 데이터사이언스, AI, 정보, 소프트웨어 |
| BOOK-B | 반도체공학과   | 전자, 전기전자, 기계, 신소재, 물리  |
| BOOK-C | 생명공학과    | 생명과학, 의예, 간호, 약학, 식품   |
| BOOK-D | 환경공학과    | 에너지, 지구, 도시, 토목, 재난안전  |
| BOOK-E | 화학과      | 화학공학, 약학, 식품, 신소재 일부   |
| BOOK-F | 경영학과     | 경제, 사회, 행정, 미디어, 정책    |

교육·예체능은 나중에 별도 작업.

---

## 5-3. 5번 작업 단계

## STEP 1. 도서 추천 큰 방향 잠금

먼저 볼 것은 다음이다.

| 항목             | 확인 기준                    |
| -------------- | ------------------------ |
| 도서가 뜨는가        | 3번·4번 선택 후 도서가 정상 노출되는지  |
| 직접 도서/확장 도서 구분 | 핵심 도서와 확장 도서가 구분되는지      |
| 엉뚱한 도서 차단      | 전공과 무관한 도서가 메인으로 뜨지 않는지  |
| 추천 이유          | 왜 이 책인지 설명이 가능한지         |
| 선택값 반영         | 개념/키워드/후속축이 도서 추천에 반영되는지 |

---

## STEP 2. 직접 도서와 확장 도서 구분

도서는 반드시 아래처럼 구분해서 봐야 한다.

| 구분    | 의미                                           |
| ----- | -------------------------------------------- |
| 직접 도서 | 선택한 교과 개념·추천 키워드·후속축을 보고서의 핵심 근거로 설명할 수 있는 책 |
| 확장 도서 | 사회적 의미, 윤리, 한계, 진로 확장에 쓰는 참고 도서              |

예시:

| 계열       | 직접 도서 방향              | 확장 도서 방향            |
| -------- | --------------------- | ------------------- |
| 컴퓨터/데이터  | 알고리즘, 데이터, 모델링, 정보 구조 | 기술 윤리, 감시사회, 플랫폼 사회 |
| 반도체/전자   | 물리, 전자, 측정, 소재, 소자    | 기술 사회 영향, 산업 변화     |
| 생명/의학    | 생명 현상, 진단, 질병, 생체 시스템 | 의료 윤리, 보건 정책        |
| 환경/에너지   | 기후, 생태, 에너지, 탄소중립     | 문명, 정책, 사회적 책임      |
| 화학/약학/식품 | 분자, 반응, 의약, 물성, 영양    | 안전성, 규제, 사회적 활용     |
| 경영/사회    | 시장, 의사결정, 통계, 소비자     | 공정, 사회구조, 윤리        |

---

## STEP 3. 선택 도서 mini payload 확인

5번에서 가장 중요한 최종 확인은 아래다.

```text
학생이 선택한 도서의 제목, 추천 이유, 보고서 활용 방식이
mini payload로 정확히 넘어가는가?
```

화면에 책이 뜨는 것만으로는 완료가 아니다.
선택한 책이 최종 보고서에 반영되는지까지 확인해야 한다.

---

## 5-4. 5번 작업의 1차/2차/3차 구분

| 단계    | 목표            | 내용                               |
| ----- | ------------- | -------------------------------- |
| 5번 1차 | 도서 추천 큰 방향 잠금 | 엉뚱한 도서 차단, 직접/확장 구분              |
| 5번 2차 | 도서별 세부 변별도    | 추천 키워드별 도서 차이, 추천 이유 보정          |
| 5번 3차 | mini/보고서 반영   | 선택 도서가 payload와 보고서 문단에 반영되는지 확인 |

```text
3. 3번·4번 선택값을 도서 추천 로직이 받고 있는지 확인
4. BOOK-A 컴퓨터/데이터 계열 기대값 제시
5. 필요 시 최소 패치 파일만 제공
```

## 7. 새 채팅에 올릴 파일

새 채팅에 올릴 것은 아래다.

```text
1. 최신 keyword-engine.zip
   - 반드시 지금까지 받은 모든 패치가 적용된 최신 프로젝트 ZIP이어야 함
   - 처음 올렸던 원본 ZIP이면 안 됨

2. 이 인계 문서 내용
   - 새 채팅 첫 메시지에 그대로 붙여넣기
```

가능하면 파일명은 짧게 한다.

```text
keyword-engine.zip
handoff.md
```

## 8. 다음 채팅 첫 요청 문장 예시

새 채팅에서 이렇게 시작하면 된다.

```text
수행평가 탐구엔진 keyword-engine 이어서 진행한다.

첨부한 ZIP은 3번·4번 1차 잠금이 모두 반영된 최신 프로젝트 파일이다.

현재 상태:
- 3번 추천 개념 대표 3개 계열별 1차 잠금 완료
- 4번 교과 개념 기반 후속 연계축 계열별 1차 잠금 완료
- 1순위 라벨 제거 완료
- 학과 라우팅, 보고서 구조, mini worker 구조, 도서 추천 구조는 아직 건드리지 않음
- 교육·예체능은 나중에 별도 작업

이번 새 채팅의 목표:
5번 도서 추천 작업으로 들어간다.

작업 방식:
1. 먼저 도서 관련 파일 구조를 확인한다.
2. 전체 구조를 갈아엎지 않는다.
3. BOOK-A 컴퓨터/데이터 계열부터 1차 검수한다.
4. 현재 210권 도서 데이터 안에서 직접 도서/확장 도서 구분이 맞는지 본다.
5. 3번 개념, 추천 키워드, 4번 후속축이 도서 추천에 반영되는지 확인한다.
6. 이상한 도서가 뜨는 부분만 최소 패치한다.
7. 패치 파일은 전체 파일이 아니라 수정 필요한 파일만 제공한다.
```

## 9. 최종 결론

현재 프로젝트 상태는 다음과 같다.

```text
3번·4번 1차 잠금 완료.
다음 단계는 5번 도서 추천 1차 검수.
```

다음 작업은 이렇게 시작하면 된다.

```text
BOOK-A 컴퓨터/데이터 계열
→ 도서 추천 로직 확인
→ 직접 도서/확장 도서 구분 확인
→ 선택값이 도서 추천에 반영되는지 확인
→ 최소 패치
```

새 채팅에서는 **5번 도서 추천부터 바로 시작**하면 된다.
