## 1. 절대 지켜야 할 작업 원칙

이번 작업의 핵심은 단순히 책 제목을 바꾸는 것이 아니다.

엔진 구조는 반드시 아래 순서를 따른다.

```text
3번 교과 개념
→ 4번 교과 개념 기반 후속 연계축
→ 5번 직접 일치 / 확장 참고 도서
→ 추천 이유
→ 보고서 활용 문장
```

도서 추천은 반드시 실제 화면에 존재하는 **3번 교과 개념명**과 **4번 후속 연계축명** 기준으로 잡아야 한다.

앞선 채팅에서 문제가 생긴 이유:

```text
1. 실제 화면 축명이 아닌 임의 축명을 기준으로 패치한 적 있음
2. 3번 개념과 4번 축을 정확히 읽지 않고 도서만 먼저 수정한 적 있음
3. 마지막 v144는 누적 패치가 아닌 원본 ZIP 기준으로 만들어졌을 가능성이 있음
```

따라서 새 채팅에서는 아래 순서를 반드시 지켜야 한다.

```text
1. 최신 keyword-engine.zip 압축 해제
2. 실제 파일에서 3번 교과 개념 후보 확인
3. 실제 파일에서 4번 후속 연계축 후보 확인
4. 화면 캡처와 파일 내부 데이터가 일치하는지 확인
5. 그다음 5번 도서 매칭만 수정
6. 패치는 필요한 파일만 ZIP으로 제공
```

## 8. 마지막 주의사항

마지막에 제공된 `book_patch_v144_geometry_computer_c_book_fix.zip`은 **최종본으로 신뢰하지 말 것**.

이유:

```text
채팅이 헤비해진 상태에서 파일 경로/누적 상태가 꼬였고,
기존 누적 패치가 포함되지 않은 원본 기준으로 만들어졌을 가능성이 있음.
```

따라서 새 채팅에서는:

```text
1. 최신 keyword-engine.zip 다시 업로드
2. 이 인계문 붙여넣기
3. 먼저 파일 내부 실제 3번·4번·5번 로직 확인
4. 기하+컴퓨터공학과 C 도서 매칭만 새로 패치
```

이 순서로 진행해야 한다.
