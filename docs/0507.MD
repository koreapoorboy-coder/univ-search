## 4. 절대 지켜야 할 작업 원칙

```text
1. 한 번에 전체 도서 데이터를 수정하지 않는다.
```

특히 중요:

```text
실제 3번 추천 키워드명
```

## 5. 현재 도서 작업의 핵심 목표

5번 도서 추천은 단순히 좋은 책을 추천하는 기능이 아니다.

기준은 다음이다.

```text
학생이 선택한 교과 개념·추천 키워드·4번 후속 연계축을
보고서에서 설명할 수 있게 해주는 책을 추천해야 한다.
```

즉, 도서는 독후감용이 아니라 **보고서 근거용**이다.

도서 추천 흐름은 다음과 같아야 한다.

```text
학과/계열
→ 과목
→ 3번 교과 개념
→ 3번 추천 키워드
→ 4번 후속 연계축
→ 직접 일치 도서
→ 확장 참고 도서
→ 보고서 활용 방식
```

## 6. 직접 일치 도서 / 확장 참고 도서 기준

도서는 반드시 두 종류로 구분한다.

```text
직접 일치 도서:
선택한 교과 개념, 추천 키워드, 4번 후속 연계축을
보고서의 핵심 원리나 분석 근거로 설명할 수 있는 책

확장 참고 도서:
사회적 의미, 윤리, 진로, 쟁점, 한계, 정책, 미래 변화 등으로 확장할 때 쓰는 책
```

예시:

```text
컴퓨터공학과 + 대수 + 데이터 예측
직접 일치 도서:
데이터, 알고리즘, 예측, 모델링, 통계적 판단, 수학적 사고와 관련된 책

확장 참고 도서:
정보사회, 감시사회, 기술 윤리, 미디어 환경, 플랫폼 사회와 관련된 책
```

## 8. 일부 겹침은 정상

4번 후속축마다 책이 완전히 달라야 하는 것은 아니다.

정상:

```text
축마다 1권 정도 공통 도서가 겹침
도서 순서가 일부 바뀜
직접 일치/확장 참고 구분이 바뀜
추천 이유와 활용 방식이 달라짐
```

비정상:

```text
4번 축 3개가 직접 일치 도서 3권까지 완전히 동일함
직접/확장 구분도 동일함
추천 이유도 동일함
```

## 9. 오른쪽 선택 도서 요약 구조

오른쪽 도서 요약은 아래 구조로 유지한다.

```text
1. 이 책은 어떤 책인가
2. 이 책을 왜 쓰는가
3. 읽을 때 확인할 부분
4. 보고서에서 활용할 수 있는 방식
5. 이 책의 핵심 키워드
6. 관련 교과
7. 관련 학과
```

이 구조는 현재 유지한다.

수정할 때는 다음 기준을 적용한다.

```text
- 내담자가 책을 읽지 않아도 책의 성격을 이해할 수 있어야 함
- 왜 이 탐구에 이 책이 필요한지 보여줘야 함
- 보고서 어느 문단에 쓸 수 있는지 보여줘야 함
- 추천 이유가 너무 추상적이면 안 됨
```

새 채팅에서 아래처럼 시작하면 된다.

```text
수행평가 탐구엔진 keyword-engine 이어서 진행한다.

첨부한 파일은 DATABASE.MD와 v101까지 적용된 최신 keyword-engine.zip이다.

현재 상태:
- 과목 선택 후 학과 검색 버퍼링 문제는 정리됨.
- 대수 + 컴퓨터공학과 기준 3번 추천 개념 로직은 정리됨.
- 4번 후속 연계축 후보가 빈공간 클릭이나 재클릭으로 바뀌는 문제는 정리됨.
- 책 클릭 시 왼쪽 목록이 바뀌는 문제도 정리됨.
- 이제 5번 도서 추천을 하나씩 검수·잠금하는 단계다.

중요 원칙:
- 실제 화면에 뜨는 3번 추천 키워드명과 4번 후속 연계축명 기준으로만 작업한다.
- 설명용 키워드나 추정 축명으로 패치하지 않는다.
- 한 번에 전체 도서를 수정하지 않고, 한 세트씩 파일을 주고 내가 화면에서 확인한 뒤 다음으로 넘어간다.
- 전체 구조, 학과 라우팅, mini worker, 보고서 구조는 건드리지 않는다.

첫 작업:
컴퓨터공학과 + 대수 + 지수함수와 로그함수의 활용 기준으로,
실제 화면에 뜨는 3번 추천 키워드와 4번 후속축을 먼저 확인한 뒤,
5번 직접 일치 도서와 확장 참고 도서가 축별로 다르게 나오도록 하나씩 패치해줘.
```

## 12. 다음 작업의 권장 순서

새 채팅에서는 아래 순서로 진행한다.

### STEP 1. 실제 화면 기준값 확인

먼저 사용자에게 아래를 확인시킨다.

```text
학과: 컴퓨터공학과
과목: 대수
3번 교과 개념: 지수함수와 로그함수의 활용
```

그다음 실제 화면에 뜨는 추천 키워드를 확인한다.

예:

```text
채널 용량
로그모델
지수모델
실생활 적용
데이터 예측
```

실제 화면에 뜨는 값만 기준으로 사용한다.

---

### STEP 2. 추천 키워드 하나 선택

예:

```text
3번 추천 키워드: 채널 용량
```

그때 실제 4번 축 3개를 확인한다.

사용자 확인 기준 예시:

```text
변화 모델링축
로그 스케일 해석 축
데이터 예측 축
```

---

### STEP 3. 4번 축별 도서 출력 확인

각 축을 하나씩 누르고 5번 직접 일치 도서 3권을 확인한다.

확인할 것:

```text
1. 변화 모델링축의 직접 일치 도서 3권
2. 로그 스케일 해석 축의 직접 일치 도서 3권
3. 데이터 예측 축의 직접 일치 도서 3권
4. 확장 참고 도서
5. 오른쪽 선택 도서 요약
```

---

### STEP 4. 한 세트만 패치

예:

```text
컴퓨터공학과 + 대수 + 지수함수와 로그함수의 활용 + 채널 용량
```

이 한 세트만 축별 도서군을 분화한다.

---

### STEP 5. 사용자 화면 확인

사용자가 정상이라고 하면 다음 키워드로 넘어간다.

정상 기준:

```text
4번 축 3개가 직접 일치 도서 3권까지 모두 동일하지 않음
축별로 도서 순서나 구성, 직접/확장 구분이 달라짐
책 클릭 시 왼쪽 목록 유지
오른쪽 요약만 변경
```

## 15. mini payload는 아직 최종 잠금 전

5번 도서 추천이 화면에서 정상 작동한 뒤에는 mini payload를 확인해야 한다.

도서 선택 이후 payload에 들어가야 할 값:

```text
department
subject
selectedConcept
selectedRecommendedKeyword
followupAxis
selectedBookTitle
selectedBookReason
selectedBookUsage
```

현재는 payload 최종 검수 전 단계다.

우선순위:

```text
1. 도서 추천 화면 안정화
2. 직접 일치/확장 참고 도서 분화
3. 오른쪽 요약 내용 보정
4. 그 다음 mini payload 반영 확인
5. 마지막으로 최종 보고서 반영 확인
```

## 17. 현재 결론


작업 방식:

```text
한 세트 패치

→ 정상 잠금
→ 다음 세트 진행
```
