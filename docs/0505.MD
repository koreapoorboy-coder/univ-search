## 수행평가 탐구엔진 keyword-engine 새 채팅 인계 문서

## 0. 현재 상황 요약

`keyword-engine` 수행평가 탐구엔진 작업을 이어서 진행한다.

현재 프로젝트는 이미 다음 단계까지 완료된 상태다.

```text
1. 희망 진로/학과 선택
2. 과목 선택
3. 교과 개념 추천
4. 교과 개념 기반 후속 연계축 선택
5. 도서 추천
6. 보고서 생성
7. mini payload 전달
8. 최종 보고서 출력
```

이 중 **3번 교과 개념 추천 + 4번 후속 연계축은 1차 잠금 완료** 상태다.

현재 새로 진행 중인 단계는 **5번 도서 추천**이다.

이번 채팅에서는 **BOOK-A 컴퓨터/데이터 계열 도서 추천 1차 검수와 UI 보정**을 진행했다.

## 2. 현재까지 고정된 프로젝트 철학

이 엔진은 단순 키워드 추천기나 주제 생성기가 아니다.

핵심 정체성은 다음이다.

```text
이 엔진은 전공 키워드 추천기가 아니라,
교과 개념에서 출발해 횡단·종단·교과 간 연결을 만들어주는
수행평가 탐구 네비게이션이다.
```

중요한 원칙:

```text
1. 전공 키워드보다 교과 개념이 먼저다.
2. 학생에게 정답 주제를 주는 것이 아니라 선택 경로를 제공해야 한다.
3. 도서는 독후감용이 아니라 보고서 근거용이다.
4. mini payload는 학생의 선택 경로가 최종 보고서에 반영되도록 하는 핵심 연결부다.
5. 수정은 항상 최소 패치로 한다.
6. 학과 라우팅, 보고서 구조, mini worker, 전체 UI 구조는 함부로 건드리지 않는다.
```

## 5. 현재 도서 추천 기준

현재 도서 추천은 “좋은 책 추천”이 아니다.

도서 추천 기준은 다음이다.

```text
학생이 선택한 교과 개념·추천 키워드·후속 연계축을
보고서에서 설명할 수 있게 해주는 책을 추천한다.
```

즉, 기준은 책의 유명도가 아니라 **보고서 활용 가능성**이다.

도서 추천 흐름:

```text
학과/계열
→ 과목
→ 3번 교과 개념
→ 추천 키워드
→ 4번 후속 연계축
→ 직접 도서 / 확장 도서
→ 보고서 활용 방식
```

## 6. 직접 도서 / 확장 도서 기준

도서는 반드시 두 종류로 나눈다.

```text
직접 도서:
선택한 교과 개념·추천 키워드·후속 연계축을
보고서의 핵심 근거로 설명할 수 있는 책

확장 도서:
사회적 의미, 윤리, 진로, 한계, 쟁점, 비교 관점으로 확장할 때 쓰는 책
```

예시:

```text
컴퓨터공학과 + 데이터 예측 축

직접 도서:
- 데이터
- 알고리즘
- 모델링
- 수학적 사고
- 통계적 판단
- 정보 구조

확장 도서:
- 기술 윤리
- 감시사회
- 미디어 환경
- 정보사회
- 플랫폼 사회
```

중요 기준:

```text
1984는 컴퓨터공학에서 쓸 수는 있지만,
데이터 예측의 직접 핵심 도서가 아니라
감시사회·정보윤리 확장 도서로 보는 것이 맞다.
```

## 10. 오른쪽 선택 도서 요약의 작성 기준

오른쪽 도서 요약은 단순 요약이면 안 된다.

내담자가 책을 읽지 않아도 아래를 파악할 수 있어야 한다.

```text
1. 이 책이 정확히 어떤 성격의 책인지
2. 왜 이 탐구 보고서에 이 책을 써야 하는지
3. 읽을 때 어떤 부분을 보면 되는지
4. 보고서에서 어떤 문단이나 근거로 활용할 수 있는지
```

예시 방향:

```text
이 책은 어떤 책인가:
책의 분야, 핵심 내용, 사고 방식, 전공 연결성을 설명한다.

이 책을 왜 쓰는가:
선택한 교과 개념·추천 키워드·후속축과 연결되는 이유를 설명한다.

읽을 때 확인할 부분:
학생이 책 전체를 읽지 않아도 어떤 관점으로 읽어야 하는지 안내한다.

보고서에서 활용할 수 있는 방식:
보고서의 원리 설명, 자료 해석, 한계 논의, 비교 분석, 후속 탐구 방향 중 어디에 쓸 수 있는지 제시한다.
```

## 13. 현재 남은 문제

현재 남은 핵심 문제는 다음이다.

```text
책을 선택한 뒤,
4번 영역 근처의 “다른 교과 개념 보기” 또는 “교과 개념 변경” 관련 버튼을 누르면
4번 후속 연계축이 바뀌는 문제가 있다.
```

사용자 확인 내용:

```text
책은 이제 클릭해도 변경되지 않는다.
그런데 책을 누른 다음 4번에서 교과 개념의 변경을 누르면
위의 4번 후속 연계축이 변경되는 상황이 발생한다.
```

정상 동작은 아래다.

```text
책 선택:
→ 5번 오른쪽 선택 도서 요약만 변경
→ 3번 교과 개념 유지
→ 4번 후속 연계축 유지
→ 5번 도서 목록 유지

다른 교과 개념 보기 버튼 클릭:
→ 3번 교과 개념 목록만 펼침
→ 기존 선택된 4번 후속 연계축 유지
→ 기존 5번 도서 목록 유지
→ 기존 선택 도서 유지
```

반대로 아래는 정상 초기화다.

```text
실제로 새로운 교과 개념을 선택함:
→ 4번 후속축 초기화
→ 5번 도서 초기화

실제로 추천 키워드를 새로 선택함:
→ 4번 후속축 초기화
→ 5번 도서 초기화

실제로 4번 후속 연계축을 새로 선택함:
→ 5번 도서 초기화
```

즉, 핵심은 이것이다.

```text
목록을 펼치는 행동과 실제 선택값을 바꾸는 행동을 분리해야 한다.
```

```text
v6 패치 파일은 다운로드 실패했으므로 실제 적용된 것으로 간주하면 안 된다.

```

현재 실제 적용된 것으로 볼 수 있는 상태:

```text

- 책 클릭 시 도서 목록 변경 문제 v5 수준까지는 해결된 것으로 보임
```

아직 미해결:

```text
- 책 선택 후 다른 교과 개념 보기/변경 버튼을 누르면 4번 후속 연계축이 바뀌는 문제
```
