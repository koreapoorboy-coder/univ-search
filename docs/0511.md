이번 채팅에서 반복적으로 발생한 오류는 **내가 확인 로직을 실제 화면과 다르게 준 것**이었다.

### 2-3. 4번 후속 연계축은 카드 제목 기준

4번 카드 안에는 제목과 설명문이 함께 있다.
그런데 도서 로직이 설명문 안의 단어를 먼저 읽으면 오작동한다.

예시 오류:

```text

설명문 안쪽: 시민 참여, 제도 분석 문구 포함
오류 결과: 감시와 처벌 / 리바이어던 / 국가 출력
```

해결 기준:

```text
4번 축 판별은 카드 설명문이 아니라 카드 제목을 우선한다.
```

### 2-4. 패치 파일은 반드시 ZIP으로 제공

```text
ZIP 내부 경로:
assets/js/book_210_ui_bridge.js
```

전체 `keyword-engine.zip`은 사용자가 명시적으로 요청하지 않으면 주지 않는다.

위 목록은 “후보”일 뿐이다.
새 채팅에서는 반드시 실제 데이터에 있는 학과만 선택해야 한다.

## 6. 새 채팅에서 바로 시작할 문장

새 채팅 첫 메시지에 아래 문장을 붙여 넣으면 된다.

```text
수행평가 탐구엔진 keyword-engine 5번 도서 추천 로직 검수 작업을 이어서 진행한다.

현재 최신 버전은 v181이고, 수정 파일은 assets/js/book_210_ui_bridge.js이다.

이번 채팅에서는 경영학과, 경제학과, 사회학과, 행정학과, 정치외교학과, 법학과, 미디어커뮤니케이션학과, 국어국문학과, 심리학과, 교육학과, 사회복지학과, 문화콘텐츠학과, 광고홍보학과, 사학과까지 완료 잠금 처리했다.

절대 규칙:
1. 학과명은 반드시 업로드한 keyword-engine.zip 내부 실제 데이터에 존재하는 학과명만 사용한다.
2. 3번 교과 개념은 실제 화면에 뜨는 값만 사용한다.
3. 4번 후속 연계축은 카드 설명문이 아니라 카드 제목을 우선해서 판별한다.
4. 임의 학과명, 임의 교과 개념명, 임의 축명을 예시로도 사용하지 않는다.
5. 계열 묶음 패치 시에는 대표 학과 1개만 전체 확인하고, 나머지는 실제 존재 학과명 기준으로 핵심 축 2~3개만 표본 확인한다.
6. 전체 파일이 아니라 적용 파일만 담은 패치 ZIP으로 제공한다.
7. ZIP 내부 경로는 assets/js/book_210_ui_bridge.js로 유지한다.
8. ACCESS_GATEWAY, mini worker, 보고서 생성 구조, UI 구조는 건드리지 않는다.

먼저 최신 keyword-engine.zip 내부에서 아직 남은 인문·사회·상경·교육·예체능 관련 실제 학과명을 확인한 뒤, 다음 대표 확인 학과 1개와 표본 확인 학과를 제시해줘.
```

## 7. 새 채팅 작업 방식

새 채팅에서는 이렇게 진행하면 된다.

```text
1. 최신 keyword-engine.zip 압축 해제
2. 실제 학과명 목록 확인
3. 이미 완료된 학과 제외
4. 남은 학과 중 대표 확인 학과 1개 선정
5. 실제 3번 교과 개념 확인
6. 실제 4번 후속 연계축 확인
7. 5번 도서가 맞는지 검수
8. 문제 있으면 book_210_ui_bridge.js만 최소 패치
9. 적용 파일만 ZIP으로 제공
10. 사용자가 화면 확인
11. 완료면 잠금 후 다음 학과 진행
```

## 8. 현재 문제 상황 메모

사용자가 마지막에 말한 상황:

```text
지금 데이터가 안 나옴
채팅이 헤비해진 것 같음
```

따라서 이 채팅에서 더 진행하지 말고 새 채팅으로 이동하는 것이 맞다.

새 채팅에서 가장 먼저 해야 할 것은 새 패치가 아니라:

```text
1. 최신 ZIP 내부 실제 데이터 확인
2. 남은 학과 목록 확인
3. 다음 대표 학과 선정
```

이다.
