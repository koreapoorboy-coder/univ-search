# NEW_CHAT_HANDOFF_2026-05-02_V79_DATA_CONNECTION_LOCK

작성일: 2026-05-02  
프로젝트: `keyword-engine` / 수행평가 탐구엔진  
문서 목적: 새 채팅에서 오늘 작업을 다시 설명하지 않고, **보고서 구조는 유지한 채 데이터 연결성만 정밀 보정**하기 위한 단일 인계 문서

---

## 0. 최종 결론

오늘 작업의 최종 결론은 아래다.

```text
1. 수행평가 탐구엔진의 화면 구조와 보고서 생성 구조는 이미 잡혀 있다.
2. 지금 해야 할 일은 구조 개편이 아니라 데이터 연결 오류 정밀 보정이다.
3. 학과 검색, 3번 교과 개념·키워드, 4번 후속 연계축, 도서, 보고서 결과가 서로 이어지는 데이터 흐름을 하나씩 검수해야 한다.
4. 오늘 여러 차례 fast guard 방식으로 학과 검색을 고치려 했지만, 오히려 구조가 꼬였다.
5. 최종적으로 학과 검색은 `major_search_fast_guard.js` 없는 native major search 방식으로 롤백했다.
6. 다음 채팅에서는 절대로 학과 검색 UI나 보고서 UI를 다시 새로 만들지 말고, 최신 zip 내부 상태를 확인한 뒤 데이터 연결성만 수정해야 한다.
```

가장 중요한 문장:

```text
이제부터는 '틀 수정'이 아니라 '틀 안에 들어가는 데이터 연결성 보정' 작업이다.
```

## 1. 이 문서가 필요한 이유

이번 세션에서는 v59부터 v79까지 많은 패치가 생성되었다. 그러나 실제로는 여러 시도 중 상당수가 최종 방향과 맞지 않아 롤백되었다.

문제가 된 흐름은 아래다.

```text
학과 검색이 느림
→ major_search_fast_guard.js 추가
→ 검색은 일부 빨라졌으나 후보 카드 UI가 생김
→ 학생이 학과를 한 번 더 클릭해야 함
→ 4번 이후 다시 학과를 바꾸면 멈춤
→ 3번·4번 상태 초기화와 검색 가드가 충돌
→ 결국 원래 native major search 방식으로 롤백
```

따라서 다음 채팅에서는 이전 패치 이름만 보고 이어가면 안 된다.  
반드시 아래 기준으로 시작해야 한다.

```text
현재 최종 검색 구조 = v79 native major search rollback
major_search_fast_guard.js = 사용하지 않음
보고서 구조 = 유지
데이터 연결성 = 정밀 보정 대상
```

## 2. 반영한 기존 인계 자료

이번 문서는 다음 기존 자료의 내용을 흡수해 하나로 정리한 것이다.

```text

```

기존 자료의 핵심은 다음과 같다.

```text
- 프로젝트는 학생이 과목과 학과를 선택하면 교과 개념, 추천 키워드, 후속 연계축, 도서, 보고서 방식, MINI 결과로 이어지는 엔진이다.
- 학생이 학과를 잘 몰라도 학과 검색으로 진입해야 한다.
- 현재 과목에서 끝나는 것이 아니라 다음 학년 과목으로 종단 확장되어야 한다.
- 추천 데이터는 사람이 보기 쉬우면서도 MINI payload로 변환 가능해야 한다.
- 교과서 데이터는 설명문이 아니라 엔진용 근거 데이터다.
- 기존 v34 인계 문서는 삭제하지 않고 구조·히스토리 기준으로 유지한다.
- v58 기준에서 4번 후속 연계축 로직이 흔들렸고, v59는 4번 후속 연계축 산출부만 최소 수정하려는 시도였다.
```

## 3. 절대 유지해야 하는 엔진 흐름

현재 화면 흐름과 보고서 구조는 유지한다.

```text
1. 과목 선택
2. 학과 검색/선택
3. 교과 개념·키워드 선택
4. 교과 개념 기반 후속 연계축 선택
5. 직접 일치 / 확장 참고 도서 선택
6. 보고서 전개 방식 선택
7. 보고서 관점 선택
8. 보고서 라인 선택
9. 학생용 결과 생성
10. MINI/Worker 결과 출력
```

이 흐름은 건드리지 않는다.

### 절대 하면 안 되는 것

```text
- 1~8번 흐름을 다시 재배치하기
- 학과 검색 UI를 완전히 새 구조로 바꾸기
- 후보 카드를 강제로 띄워 한 번 더 클릭하게 만들기
- 6~8번을 운영자용 payload 화면으로 바꾸기
- 학생이 보고서 방향을 선택하지 못하게 만들기
- 학과만 보고 보고서를 생성하기
- 3번 교과 개념과 4번 후속 연계축을 건너뛰기
- 도서를 마지막 카드에만 반영하기
- 결과 생성부만 반복 패치하기
```

### 현재 작업의 정확한 범위

```text
수정 대상:
- 학과 검색 결과값 / alias / 우선순위
- 3번 교과 개념 추천 순서
- 3번 추천 키워드 묶음
- 4번 후속 연계축 1순위
- 직접 연계 강함 / 역량 브리지 / 보조 확장 판정
- 도서 추천 연결
- 최종 보고서에 들어가는 학과·교과 관점 문장

수정 금지 대상:
- 화면 틀
- 보고서 구조
- 1~8번 진행 순서
- MINI Worker 구조
- 새로운 검색 가드 스크립트 추가
```

## 7. 오늘의 실패 원인 정리

이번 세션에서 진도가 늦어진 이유는 아래다.

### 7-1. 구조 문제와 데이터 문제를 섞어서 봄

처음 목표는 4번 후속 연계축 데이터 보정이었는데, 중간에 학과 검색 버퍼링과 UI 문제를 함께 잡으면서 작업 범위가 커졌다.

다음부터는 아래처럼 분리해야 한다.

```text
1. 학과 검색 구조 문제
2. 3번 교과 개념 추천 문제
3. 4번 후속 연계축 데이터 문제
4. 도서 연결 문제
5. 보고서 결과 문장 문제
```

하나씩만 수정해야 한다.

---

### 7-2. major_search_fast_guard.js를 추가한 것이 부작용을 만듦

fast guard는 검색 반응을 빠르게 하려고 만든 우회 로직이었지만, 기존 엔진의 상태 관리와 충돌했다.

문제:

```text
- 별도 후보 카드 생성
- 한 번 더 클릭해야 함
- 학과 확정 시점이 불명확해짐
- 3번/4번 재계산 시점이 꼬임
- 4번 이후 학과 재검색 시 멈춤
```

다음 작업자 주의:

```text
검색 버퍼링이 다시 보이더라도 major_search_fast_guard.js 방식으로 되돌아가지 말 것.
```

---

### 7-3. 4번을 먼저 볼지, 학과 검색을 먼저 볼지 순서가 흔들림

학과 검색이 잘못 잡히면 4번이 틀리는 것이 당연하다.  
하지만 학과 검색 UI를 과하게 바꾸면 또 전체 구조가 흔들린다.

다음 기준:

```text
1. 학과 검색은 native 구조 유지
2. 검색 결과가 틀리면 major_engine_helper 내부 alias/score만 보정
3. 3번·4번은 textbook_concept_helper와 followup data에서 보정
4. 검색 UI는 바꾸지 않는다
```

---

### 7-4. 패치가 너무 많은 파일을 건드림

앞으로 패치는 반드시 최소 파일만 포함한다.

```text
학과 검색 오류
→ major_engine_helper.js만 우선 확인
→ 필요 시 index.html 캐시 버전만 변경

3번 교과 개념/키워드 오류
→ textbook_concept_helper.js 또는 교과 seed만 확인

4번 후속 연계축 오류
→ textbook_concept_helper.js / major_followup_axis.json 확인

도서 오류
→ book_recommendation_adapter.js / book data 확인

결과 문장 오류
→ mini_worker_generate_bridge_v32.js 또는 mini_payload_builder.js 확인
```

## 10. 현재 잠금된 개념: 횡단 평가와 종단 연결

### 10-1. 횡단 평가

횡단 평가는 같은 시기 안에서 교과 간 연결성이 보이는 구조다.

예시:

```text
통합과학에서 폭염주의보를 다룸
→ 통합사회에서 취약 계층·생활환경 문제와 연결
→ 수학에서 자료 비교·그래프 해석과 연결
→ 정보에서 입력값·조건문·판단 기준과 연결
```

보고서에는 아래 관점으로 반영된다.

```text
자료 해석
사회적 의미
수치 비교
과학적 근거
생활 문제 적용
```

### 10-2. 종단 연결

종단 연결은 현재 교과 개념이 다음 학년 과목이나 더 심화된 과목으로 이어지는 흐름이다.

예시:

```text
통합과학1: 과학의 측정과 우리 사회
→ 수학: 대수, 확률과 통계
→ 정보: 데이터 처리, 알고리즘
→ 지구과학: 기후·대기 자료 해석
→ 물리학: 측정 장치, 시스템 해석
```

4번 후속 연계축은 기본적으로 종단 연결 장치다.

### 10-3. 4번의 정확한 역할

```text
교과 개념 기반 종단 연결
+
학과 기반 우선순위 보정
+
보고서 방향의 사고축 결정
```

4번은 아래 세 가지를 동시에 만족해야 한다.

```text
1. 교과 개념에서 출발해야 한다.
2. 다음 과목으로 이어지는 종단성을 보여야 한다.
3. 학과에 따라 1순위 축과 설명 문장이 달라져야 한다.
```
