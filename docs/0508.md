## 2. 절대 지켜야 할 작업 원칙

이번 채팅에서 가장 중요한 교훈은 이것이다.

```text
3번 추천 키워드와 4번 후속축은 절대 추정해서 만들면 안 된다.
반드시 실제 데이터/화면에 뜨는 값 기준으로 도서 연결을 해야 한다.
```

앞에서 문제가 되었던 사례:

```text
확률과 통계 + 조건부확률과 사건의 독립에서

- 독립성 검증 축
- 조건부 판단·리스크 해석 축

조건부확률 구조 축 / 사건 관계 분석 축 / 확률 논리 모델링 축
```

```text

6. 틀린 축만 최소 패치
```

## 3. 작업 방식 변경 사항

## 12. 현재 화면 검수 기준

앞으로 콘솔보다 화면 직접 검수 방식으로 간다.

```text

4. BOOK-A 같은 내부 라벨이 보이면 안 됨
5. 도서 오른쪽 요약에 내부 코드명이 보이면 안 됨
6. 보고서 생성 시 선택한 과목, 개념, 축, 도서가 반영되어야 함
```

보고서 반영 화면 검수 기준:

```text
예: 미적분1 + 도함수의 활용 + 운동 변화율·기계 모델링 축 + 카오스

- 미적분1
- 도함수의 활용
- 운동 변화율
- 기계 모델링
- 카오스
- 변화율
- 모델링
```

오류로 보는 경우:

```text
- 과목이 정보로 잘못 나옴
- 선택 도서가 빠짐
- 후속축이 빠짐
- “선택한 도서”라고만 나오고 책 제목이 없음
- 보고서가 전공/과목 맥락과 다르게 나옴
```
